# Data Card — Stablecoin Contagion GNN Benchmark

## Overview

This dataset supports the study of cross-stablecoin depeg contagion.
Each instance is a (node, time) pair labelled with whether the node
enters a new stress episode within a fixed prediction horizon.

## Node universe

| Dimension | Values |
|---|---|
| Assets (fiat-backed) | USDC, USDT, TUSD, PYUSD, BUSD |
| Assets (crypto-backed) | DAI, FRAX |
| Assets (synthetic) | USDe |
| CEX venues | Binance, Coinbase, Kraken |
| DEX venues | Curve (3pool, FRAXUSDC, USDp, sUSDv2), Uniswap v3 |

A **node** is an (asset, venue, fee_tier) triple.

## Data sources

| Source | Data | Granularity | Access |
|---|---|---|---|
| Binance / Coinbase / Kraken via CCXT | 1-min OHLCV | 1 min | Free (rate-limited) |
| Uniswap v3 subgraph (The Graph) | Pool TVL, volume | Hourly | Free (API key recommended) |
| DeFiLlama | Curve pool TVL | Daily | Free |

**Historical coverage caveat**: 1-min Binance data is fully available from
~2019. Kraken data for 2018 may require exchange archives or Kaiko.
The 2018 USDT episode is included but may have reduced venue coverage —
see `data/processed/availability_matrix.csv` for exact per-episode coverage.

## Label definition (pre-registered)

**Onset label** `y_{j,t} = 1` if:
1. Node j is NOT in stress at time t (stressed = |peg deviation| exceeds threshold for ≥ 10 consecutive minutes)
2. Node j ENTERS a stressed period within the prediction horizon Δ

**Stress thresholds:**
- Fiat-backed: |deviation| > **25 bps** for ≥ 10 min
- Crypto-backed: |deviation| > **75 bps** for ≥ 10 min
- Synthetic: |deviation| > **50 bps** for ≥ 10 min

These thresholds were fixed before any modelling and are reported
with a sensitivity analysis in Phase 5.

## Synthetic augmentation

200 synthetic episodes are generated by block-resampling 1-min price windows
from non-stress periods and injecting a depeg shock.
Synthetic episodes are validated against real episodes using a KS test on:
OU half-life, realized volatility (1h/24h), and price deviation moments.
Episodes that fail the validation check (p < 0.05) are discarded.

## Gap handling

- Gaps ≤ 5 minutes: forward-filled from the last valid value.
- Gaps > 5 minutes: left as NaN.
- Episodes with overall coverage < 80% are excluded (see `min_coverage_pct` in config).
- All gaps are logged to `data/raw/gaps.csv`.

## Split

Chronological only — no random shuffling at any stage.
Test set: USDC_SVB (chosen because its trigger type, `fiat_bank` from a
counterparty failure, differs from all training triggers in key ways, providing
a meaningful out-of-distribution generalization test).

## Reproducibility

```bash
make stub        # reproduce on synthetic data (no API keys)
make all         # reproduce on real data (requires CCXT + The Graph access)
```

All random seeds are fixed globally in `src/scgnn/utils/seeds.py` (seed=42).
A `dataset_manifest.json` is written to `data/processed/` recording
which episodes, features, and label horizons were used for each split.
